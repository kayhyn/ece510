Composition (architectural):
  - 128 MAC lanes (the mac_array compute core)
  - 128 per-lane weight banks (L_MAX=64 entries x 8b each, register-inferred)
  - Banked broadcast tree (8 banks of 16 lanes; 1 cycle of registered fan-out)
  - Result serializer (1 result per beat over the 64-bit AXI4-Stream output)
  - Opcode-tagged input decoder (LOAD_WEIGHT / COMPUTE)

The 84.9 k sequential cells are consistent with the architectural inventory:
128 lanes x (8b weight + 16b product + 64b CSA + 32b result + 3b control) =
~16 k inner-array flops; plus 128 x 64-deep x 8b weight banks = ~65 k flops
in the weight register files; plus broadcast pipeline and serializer = a
small remainder. The 53 k timing-repair buffers reflect the heavy fan-out
of the activation and control pipelines across 128 lanes even after the
banked broadcast tree.

----------------------------------------------------------------------------
Naive 128-lane extrapolation
----------------------------------------------------------------------------

Single-lane post-PnR x 128:
  Cell count:  2,498 x 128 = 319,744 stdcells
  Area:        13,128.8 x 128 = 1,680,486 um^2  (~1.68 mm^2)

The actual `accel_top` is smaller-than-128x-lane because:
  + shared control pipeline (one set of valid/first/last + activation regs)
  + the banked broadcast tree's CTS savings
  - but adds the 128 per-lane weight register files and serializer FSM

The authoritative full-array number is the `accel_top` post-PnR number in
section (A); the 128x lane figure is an order-of-magnitude check.

----------------------------------------------------------------------------
Comparison to predecessors
----------------------------------------------------------------------------

  M3 single 9-tap lane (M3_SYNTH, pre-PnR):  ~2,000 cells, 23,130 um^2
  M4 single lane post-PnR (with CSA):         2,498 cells, 13,129 um^2

Per-lane area DOWN despite the new CSA registers, because M3 included AXI
command glue and the runtime tap-select mux that M4 drops; M4 amortizes those
across one shared pipeline.

----------------------------------------------------------------------------
Dominant area contributors
----------------------------------------------------------------------------

In the placed lane, the dominant area contributors are:
  1. The 8x8 signed multiplier (~25% of combinational cells)
  2. The 32-bit CSA sum/carry registers (~24% of sequential area)
  3. The 32-bit final-resolve adder, used only on `in_last`
  4. The clock-tree buffers (~13 clock inverters + 19 clock buffers per lane)

In the 128-lane array, the additional dominant contributor is:
  5. The 128 per-lane weight register files (L_MAX x 8b each)
