Status
------
The production wrapper completed synthesis, placement, CTS, post-CTS timing
repair, and global routing with zero overflow. Detailed routing failed during
TritonRoute track assignment. Therefore the values below are post-CTS /
post-global-route estimates, not sign-off timing.

The final benchmark projection uses 114.536 MHz because this is the deepest
available setup result for the complete interface-connected production wrapper.
Full-wrapper slow/fast-corner sign-off results are unavailable because detailed
routing and post-PnR multi-corner STA did not complete.

The full wrapper does not close setup or hold timing at this snapshot. The
reported register-to-register hold paths are positive; the unresolved hold set
is outside register-to-register paths. Because detailed routing and final
timing repair did not complete, these violations remain unresolved and the
114.536 MHz value is a setup-limited estimate, not sign-off.

Worst setup path
----------------
Source: accel_postcts_checks.rpt

  Startpoint: result-path flip-flop `_509898_`
  Endpoint:   serialized output port `m_tdata[26]`
  Path:       result register -> fanout/max-cap buffers -> serializer mux logic
              -> output buffers -> m_tdata[26]
  Slack:      -2.730896 ns

The final full-wrapper critical path is therefore in the serialized output
interface, not in the MAC multiplier. This is consistent with the final
interface/serialization-bound benchmark.

I/O timing limitation
---------------------
The run log warns that `PNR_SDC_FILE` and `SIGNOFF_SDC_FILE` were not defined,
so OpenLane used its generic fallback SDC. The unresolved non-register-to-
register hold set must therefore be interpreted as an interface-constraint and
unfinished-repair failure, not as demonstrated host-interface closure. A
follow-on run must define explicit input/output delays for the assumed FPGA-SoC
host, complete detailed routing, and rerun setup/hold repair and sign-off STA.

Conclusion
----------
The final production chiplet does not close setup or hold timing at the
available post-CTS snapshot. Its benchmark uses the setup-limited 114.536 MHz
estimate. The separate lane run demonstrates that the multiplier is the
remaining per-lane critical path; the full wrapper adds substantial
weight-storage, clock-tree, shared-broadcast, and routing overhead.
