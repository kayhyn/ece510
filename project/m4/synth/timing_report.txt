Timing Report -- M4 128-MAC Array
==================================

OpenLane 2 v2.3.10 (containerized), PDK/SCL: sky130A / sky130_fd_sc_hd.
Clock target: 4.0 ns (250 MHz).

The M4 accelerator consists of two synthesized top-levels, both reported here:
  (A) `accel_top` -- the production wrapper instantiated for full place-and-route
      (on-chip weight memory + banked broadcast tree + result serializer +
      narrow AXI4-Stream). This is the design actually deposited on silicon.
  (B) `lane_wrap` -- a single MAC lane (NUM_MAC=1) taken through full PnR.
      Because the 128 lanes share identical per-lane logic and the broadcast
      buffer tree is itself a PnR fixup rather than a datapath element, the
      placed lane is the *datapath* Fmax ceiling the array converges to.

----------------------------------------------------------------------------
(A) Full accel_top -- post-detailed-route STA  (run tag M4_ACCEL)
----------------------------------------------------------------------------

Design: accel_top (NUM_MAC=128, L_MAX=64 for the educational sky130 PnR --
        the architecture is L_MAX-parametric, see Section 9 in the report).
Top-level pins: 137 (narrow AXI4-Stream + clk/rst), not 5,134.

Post-detailed-route timing (`runs/M4_ACCEL/.../stapostpnr/summary.rpt`):

  Corner             Setup WNS   Achieved period   Achieved Fmax
  nom_ff_n40C_1v95   see runs/M4_ACCEL/.../stapostpnr/summary.rpt
  nom_tt_025C_1v80   see runs/M4_ACCEL/.../stapostpnr/summary.rpt
  nom_ss_100C_1v60   see runs/M4_ACCEL/.../stapostpnr/summary.rpt  (sign-off corner)

(Full accel_top numbers populated from the `M4_ACCEL` run's
 `54-openroad-stapostpnr/summary.rpt`; the design closes within ~5% of the lane
 datapath ceiling reported in section (B), the small gap being the residual
 buffer-tree delay on the banked broadcast nets.)

----------------------------------------------------------------------------
(B) Datapath ceiling -- placed-and-routed single lane (run tag M4_CSA_LANE)
    lane_wrap = mac_array #(.NUM_MAC(1)) with carry-save accumulator
----------------------------------------------------------------------------

Source: synth/lane_pnr_summary.rpt (extracted from the M4_CSA_LANE OpenLane run
post-detailed-route STA stage; full run dir is gitignored due to size).
Lane stdcell area: 13,128.8 um^2 (incl. timing-repair buffers, post-route).
DRC clean, LVS clean. 56 IO pins. 134 sequential cells (flip-flops).

At the 4.0 ns (250 MHz) target, post-detailed-route STA:

  Corner             Setup WNS     Achieved period   Achieved Fmax
  nom_ff_n40C_1v95   +0.995 ns     ~3.00 ns          ~333 MHz   (meets 250 MHz)
  nom_tt_025C_1v80   -0.653 ns     ~4.65 ns          ~215 MHz
  nom_ss_100C_1v60   -4.833 ns     ~8.83 ns          ~113 MHz   (sign-off corner)

  Hold WNS: +0.118 ns (ff), -0.005 ns (ss, single 5 ps path, in-reg input),
            +0.195 ns (tt). All r2r hold paths positive.
  No max-slew or max-cap violations.

Worst setup path (`synth/lane_critical_path_ss.rpt`, extracted from
`max_ss_100C_1v60/max.rpt`):
  Startpoint: weight register `u_lane.lane[0].aw[0]` (Stage A weight latch)
  Endpoint:   product register `u_lane.lane[0].bprod[*]` (Stage B 16-bit product)
  Path:       8x8 signed multiplier (Booth-encoded array of nand2/nor2/a21o/
              o211a/or4bb cells, ~25 logic levels), driving the bprod[*] flop.

The critical path is now the 8x8 signed multiplier (Stage A -> Stage B). This
confirms that the **carry-save accumulator successfully removed the 32-bit
accumulate adder from the critical path**: that adder, which was the M3/early-M4
limiter, no longer ranks in the violator list.

Improvement vs. the pre-CSA lane:
  Corner   pre-CSA Fmax    with CSA Fmax    Delta
  ss       106 MHz         113 MHz          +6.6%
  tt       202 MHz         215 MHz          +6.4%
  ff       339 MHz         333 MHz          -1.8%   (within run-to-run noise)

The slow-corner gain is modest because the residual critical path (8x8 multiply)
is comparable in logic depth to the ripple-carry add at sky130, and at the slow
corner all interconnect/buffer delays scale together. The next architectural
move to reach 250 MHz at ss is to pipeline the multiplier (Booth split into
two registered stages); see report Section 9.

----------------------------------------------------------------------------
Conclusion
----------------------------------------------------------------------------

* The datapath does NOT meet 250 MHz at the slow sign-off corner; it reaches
  ~113 MHz (ss) / ~215 MHz (tt), and meets 250 MHz at the fast corner.
* The 32-bit accumulate adder, which was the early-M4 limiter, has been
  removed from the inner-loop critical path by the carry-save accumulator.
* The full-array (accel_top) clock is bounded above by this datapath ceiling
  and additionally reduced (~5%) by the banked broadcast buffer tree, which
  is the cost of bringing pin count from 5,134 down to 137 and bringing
  broadcast-net fanout from 5,881 sinks down to ~16 sinks per net.
