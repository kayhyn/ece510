Power Report -- M4 128-MAC Array
=================================

OpenLane 2 run:
  OpenLane v2.3.10 (containerized via colima/docker)
  Run tag: M4_SYNTH
  Command: openlane --run-tag M4_SYNTH --overwrite --to OpenROAD.STAPrePNR
           project/m4/synth/config.json
  PDK/SCL: sky130A / sky130_fd_sc_hd
  Clock target: 4.0 ns (250 MHz)

Pre-PNR power estimate (OpenROAD, default switching activity), per corner:

  Corner             Internal     Switching    Leakage      Total
  nom_tt_025C_1v80   200.67 mW    54.71 mW     0.43 uW      255.37 mW
  nom_ss_100C_1v60   149.94 mW    42.90 mW     0.61 mW      193.45 mW
  nom_ff_n40C_1v95   277.69 mW    64.84 mW     1.21 uW      342.53 mW

  tt split: Sequential 131.1 mW (51.3%), Combinational 124.3 mW (48.7%).

Comparison to the M3 single 9-tap lane (M3_SYNTH):

  M3 single lane:   ~1.569 mW total
  M4 128-MAC array: ~255 mW (tt) total

Interpretation:
This is a PRE-PNR estimate using OpenLane default activity (not VCD/SAIF
annotated from the streaming workload), and the array is 128 always-active
lanes, so the absolute value is an early upper-ish estimate. It is materially
higher than the linear 128x projection used in CF09 (~133 mW): the real array
carries full 32-bit accumulators and product/weight pipeline registers per lane
(11,279 FFs total), so sequential power alone is ~131 mW at tt. For a final
number, M4+ should rerun power with activity annotated from the tb_mac_array
streaming run and after place-and-route clock-tree synthesis.
